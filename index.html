A HTTP protokoll

A HTTP (HyperText Transfer Protocol) egy hálózati kommunikációs protokoll. A webszerver és a kliens ezen szabályok alapján bonyolítanak adatforgalmat.
A kliens, másnéven a user agent, általában véve egy böngésző, a szerver pedig egy webszerver.

A kliens kérést intéz a szerver felé

A kérés első sora metodús/erőforrás/verzió alapú

A HTTP metódus

A metódus mondja meg a kommunikáció formáját.

  • GET: egy erőforrás lekérdezése (nincs üzenettest)
  • POST: űrlap információk feltöltése
  • DELETE: erőforrás letörlése (nincs üzenettest)
  • PATCH: részleges felülírás
  • PUT: teljes felülírás
  • HEAD: visszaadja az ugynevezett fejlécállományt

Fejlécállomány

A fejlécállomány (head) a kliens és a szerver is küldhet. Ezek vagy beállítások, vagy információk a küldőről.

Erőforrás

Az erőforrás az általában véve egy url, vagy egy fájl amin kerresztül egy erőforrást el tudunk érni.

Verzió

  • HTTP/1.1 - titkosítatlan HTTP
  • HTTP/2.0 - titkosított HTTP (https)

Üzenettest (body)

Az üzenettestben a szerver felé küldött adatok találhatóak (opcionális).

HTTP válasz

A http válasz első sora verzió/státuszkód/szöveges magyarázat alakú.

Válasz státuszkód

  • 1xx - informatív válasz(mit kellenne a kliensnek a továbbiakban csinálni)
  • 2xx - minden rendben
  • 3xx - átírányítás (egy oldalnak a régi webcímét ütöm be akkor tud olyan átírányítást csinálni ami átvisz az új webcímre)
  • 4xx - kliens hiba (400:bad request, 403:forbidden, 404:nem található az erőforrás)
  • 5xx - szerver hiba(500:internal server error, 503: service unavailable)

Content-Type nevű fejléc

A content-type azt mondja meg, hogy milyen típusú adatot küldünk/fogadunk.

  • text/html - html kód
  • text/plain - egyszerű szöveg
  • application/json
  • x-www-form-urlencoded
  • multi/form-data
  • image/jpeg - image/png
